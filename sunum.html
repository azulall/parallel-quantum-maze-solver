Parallel Quantum-Inspired Maze Solver

Ders: Paralel Programlama  |  Tarih: Mayıs 2026

1. Projenin Amacı

Bu projede bir labirenti iki farklı yöntemle çözüyoruz ve performanslarını karşılaştırıyoruz:

Sequential (Ardışıl) DFS: Tek thread ile klasik Depth-First Search. Bir yoldan ilerler, çıkmaz sokağa girerse geri döner, başka yolu dener. Tüm iş TEK bir işlemci çekirdeğinde yapılır.

Parallel Quantum-Inspired Solver: OpenMP ile birden fazla thread kullanır. Aynı anda birden çok yolu araştırır. Üstelik "quantum-inspired" sıralama ile çıkışa daha yakın yolları ÖNCE dener.

NOT: Bu projede gerçek kuantum bilgisayar kullanılmamaktadır. "Quantum-inspired" ifadesi, Grover algoritmasındaki "doğru çözümün olasılığını artırma" fikrinin klasik bilgisayarda simüle edilmesi anlamına gelir.

4. Kodun Kritik Kısımları

4.1. Sequential DFS — Baseline (Karşılaştırma Temeli)

Tek thread ile çalışır. Bir yoldan gider, tıkanırsa geri döner. Basit ama yavaş.

// SequentialDfsSolver.cpp — Iteratif DFS
// Stack kullanarak yolları tek tek dener
std::stack<Frame> callStack;
callStack.push({start_x, start_y, 0});

while (!callStack.empty()) {
    Frame &cur = callStack.top();
    
    if (cur.x == end_x && cur.y == end_y) {
        return true;  // Çıkışı bulduk!
    }
    
    // 4 yönü dene: sağ, sol, aşağı, yukarı
    int nx = cur.x + dx[cur.dirIdx];
    int ny = cur.y + dy[cur.dirIdx];
    cur.dirIdx++;
    
    if (valid) {
        solution[ny][nx] = 8;  // İşaretle
        callStack.push({nx, ny, 0});  // İlerle
    }
    // Tüm yönler denendiyse: geri dön (backtrack)
}

→ Problem: Tek thread olduğu için tüm yollar SIRAYLA denenir. Büyük labirentlerde çok yavaş.

4.2. Parallel Candidate Preparation — Aday Hücrelerin Belirlenmesi

Mevcut pozisyondan gidilebilecek geçerli komşu hücreleri bulur.

// ParallelCandidates.cpp
std::vector<SearchNode> MazeSolver::getParallelCandidates(int x, int y, int depth) {
    std::vector<SearchNode> candidates;
    
    for (int i = 0; i < 4; i++) {     // 4 yön: sağ, sol, aşağı, yukarı
        int nx = x + dx[i];
        int ny = y + dy[i];
        
        if (isValid(nx, ny)) {          // Duvar değilse ve ziyaret edilmediyse
            candidates.push_back({nx, ny, depth + 1});
        }
    }
    return candidates;
}

→ Bu fonksiyon her hücrede çağrılır ve "hangi yollara gidebiliriz?" sorusunu cevaplar.

4.3. Quantum-Inspired Ordering — Akıllı Sıralama

NEDEN ÖNEMLİ? Normal DFS rastgele yönlere gider. Ama biz çıkışa YAKIN olan yolları ÖNCE denemek istiyoruz. Böylece çözümü daha hızlı buluruz. Bu fikir Grover algoritmasındaki "amplitude amplification" kavramından esinlenmiştir. 
// QuantumHeuristic.cpp

// Manhattan mesafesi ile hedefe uzaklığı ölç
// Sonra 1/sqrt(distance) formülüyle ağırlık hesapla
// Yakın olan hücre → yüksek ağırlık → önce denenecek!

double QuantumHeuristic::calculateAmplitudeWeight(Point2D nextPos, Point2D targetPos) {
    double distance = abs(nextPos.x - targetPos.x) + abs(nextPos.y - targetPos.y);
    
    if (distance == 0.0) return 999999.0;  // Hedefteyiz!
    
    return 1.0 / sqrt(distance);  // Yakın = yüksek ağırlık
}

// MazeSolver içinde kullanımı:
candidates = orderCandidatesByWeight(candidates);
// → En umut verici yollar listenin BAŞINA geçer

Örnek: Çıkışa 10 birim uzak bir hücre → ağırlık = 1/√10 = 0.316. Çıkışa 2 birim uzak bir hücre → ağırlık = 1/√2 = 0.707. İkinci yol ÖNCE denenir!

4.4. OpenMP Parallel DFS — Paralel Arama Motoru

PROJENİN KALBİ BURADA! Bu fonksiyon birden fazla yolu AYNI ANDA araştırır. Her candidate için ayrı bir OpenMP task oluşturulur ve farklı thread'ler tarafından çalıştırılır. 
// ParallelSolver.cpp — Paralel DFS motoru

bool MazeSolver::parallelDFS(int x, int y, bool& found, int depth) {
    if (found) return true;            // Başka thread buldu, dur
    if (!tryVisitCell(x, y)) return false;  // Zaten ziyaret edilmiş
    
    if (x == endX && y == endY) {      // Çıkışı bulduk!
        found = true;
        return true;
    }
    
    // 1) Gidilebilecek hücreleri bul
    auto candidates = getParallelCandidates(x, y, depth);
    
    // 2) Quantum sıralaması uygula (yakın olanlar önce)
    candidates = orderCandidatesByWeight(candidates);
    
    // 3) Her candidate için paralel task oluştur
    for (auto& node : candidates) {
        if (found) break;  // Erken sonlandırma
        
        if (shouldCreateTask(node.depth)) {
            // ↓↓↓ PARALEL KISIM: Yeni thread bu yolu araştırır ↓↓↓
            #pragma omp task shared(found) firstprivate(node)
            {
                parallelDFS(node.x, node.y, found, node.depth);
            }
        } else {
            parallelDFS(node.x, node.y, found, node.depth);
        }
    }
    
    #pragma omp taskwait  // Alt task'ların bitmesini bekle
    return found;
}

4.5. Atomic Visited Kontrolü — Thread-Safe Ziyaret

Birden fazla thread aynı hücreyi ziyaret etmeye çalışabilir. Bunu önlemek için atomic capture kullanıyoruz:

// ParallelSolver.cpp — Lock-free hücre kontrolü

bool MazeSolver::tryVisitCell(int x, int y) {
    int idx = y * cols + x;        // Düz dizi indeksi (flat array)
    int oldVal;
    
    #pragma omp atomic capture     // Donanım seviyesinde atomik işlem
    {
        oldVal = visited[idx];     // Eski değeri oku
        visited[idx] = 1;          // Ziyaret edildi olarak işaretle
    }
    
    return oldVal == 0;  // İlk ziyaret eden thread ise true döner
}
NEDEN critical YERINE atomic?
#pragma omp critical → mutex kullanır, thread'ler sıra bekler → YAVAŞ
#pragma omp atomic capture → donanım seviyesinde, kilit yok → HIZLI
Her hücre ziyaretinde çağrıldığı için bu fark çok büyük!

4.6. Task Derinlik Kontrolü

// Sığ derinliklerde task oluştur (parallellik sağla)
// Derin seviyelerde sıralı devam et (gereksiz overhead'den kaçın)

static const int TASK_DEPTH_LIMIT = 6;

bool shouldCreateTask(int depth) {
    return depth < TASK_DEPTH_LIMIT;  // Derinlik 0-5: paralel, 6+: sıralı
}

6. Neden Bu Kadar Hızlı?

Paralel solver'ın normal "10 thread = 10x hız" beklentisini aşmasının iki sebebi var:

6.1. Quantum-Inspired Ordering Etkisi

Sequential DFS rastgele yönlere gider. Paralel solver ise çıkışa yakın yolları önce dener. Bu sayede paralel solver, sequential'ın hiç bakmadığı gereksiz yolları atlar ve çözümü çok daha erken bulur.

ÖzellikSequential DFSParallel Quantum Solver
Thread sayısı110
Yön seçimiSabit sıra (sağ, sol, aşağı, yukarı)Akıllı sıra (çıkışa yakın önce)
Arama stratejisiKör aramaYönlendirilmiş arama
Erken sonlandırmaYokVar (found flag ile)
Visited kontrolüBasit diziAtomic capture (lock-free)

6.2. Parallellik + Akıllı Arama = Süper-Lineer Speedup

1001×1001'deki 75x speedup "süper-lineer" bir sonuçtur. Bunun sebebi:

  • Sequential: Yanlış yolda uzun süre ilerleyebilir, sonra geri dönüp tekrar dener
  • Parallel: 10 thread AYNI ANDA farklı yolları dener + quantum ordering ile EN DOĞRU yoldan başlar
  • Sonuç: Paralel solver, sequential'ın incelediği hücrelerin çok az bir kısmına bakarak çözümü bulur
